# Proposal: 生字本入口条件显示

## Summary

优化个人页（profile）的生字本入口显示逻辑：仅当用户有历史检测记录时才显示生字本 UI 区域，无记录时隐藏该区域。

## Motivation

当前实现中，生字本入口卡片始终显示在个人页，即使用户没有任何历史检测记录。这存在以下问题：

1. **功能相关性**：生字本数据来源于历史检测记录中的 `unknownChars`，如果没有历史记录，生字本必然为空（显示 0），展示该入口意义不大
2. **用户体验**：对于新用户，展示一个"数量为 0"的生字本入口可能造成困惑，不如引导用户先进行测试
3. **界面简洁**：减少不必要的 UI 元素，让界面更聚焦

## Proposed Changes

### 行为变更

修改个人页的生字本入口卡片显示逻辑：

| 条件 | 当前行为 | 修改后行为 |
|------|---------|-----------|
| 有历史检测记录 | 显示生字本入口 | 显示生字本入口（不变） |
| 无历史检测记录 | 显示生字本入口（数量为 0） | **隐藏**生字本入口区域 |

### 涉及范围

- **页面**：`src/pages/profile/profile.vue`
- **规范**：`openspec/specs/profile-page/spec.md`

### 技术方案

在生字本入口卡片外层添加 `v-if="records.length > 0"` 条件判断，复用现有的 `records` 响应式变量。

## Impact

- **用户影响**：新用户首次进入个人页时不会看到生字本入口，完成首次测试后该入口自动出现
- **向后兼容**：对已有记录的用户无影响
- **性能影响**：无，只是条件渲染

## Alternatives Considered

1. **始终显示，但加提示文字**：如"完成测试后解锁"，但增加了 UI 复杂度
2. **显示为禁用状态**：视觉上更复杂，且意义不大
