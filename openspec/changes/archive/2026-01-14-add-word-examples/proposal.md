# add-word-examples

## Why

当前检测页仅展示单个汉字，对于识字能力较弱的儿童，仅凭单字难以判断是否真正认识该字。通过展示包含该汉字的词语示例：
1. 帮助儿童通过词语语境理解汉字含义
2. 增强测试的教育价值，让儿童在测试中学习
3. 提升测试的趣味性和互动性
4. 词语发音功能加深对汉字读音的记忆

## What Changes

### 1. 数据源切换
- 从 `top_2500_chars_with_literacy.json` 切换到 `top_2500_chars_with_words.json`
- 新数据源包含 `words` 字段，每个汉字对应两个常用词语

### 2. UI 变更（参考 Figma 设计稿）
- 在主米字格下方添加"📚 词语示例"标题
- 添加四个小米字格（160rpx），两两一组展示两个词语
- 布局：左侧词语1（2字）+ 右侧词语2（2字）

### 3. 发音功能增强
- 微信小程序环境下，仅在用户手动点击喇叭按钮时播放发音
- 播放顺序：汉字 → 词语1 → 词语2（无间隔，连续播放）
- 汉字切换时不自动播放发音

## Scope

### In Scope
- 检测页 UI 调整，添加词语展示区域
- 切换数据源并加载词语数据
- 词语发音播放（微信小程序）
- 条件编译处理平台差异

### Out of Scope
- 词语数据的编辑或管理
- H5 或其他小程序平台的发音功能
- 词语展示的用户自定义设置

## Dependencies

- 微信同声传译插件 WechatSI（已配置）
- `top_2500_chars_with_words.json` 数据文件（已存在）
- RiceGrid 组件（复用现有组件）

## Risks & Mitigations

| 风险 | 缓解措施 |
|------|----------|
| 词语发音耗时增加 | 使用异步队列播放，不阻塞用户操作 |
| 部分汉字无词语数据 | 检查 words 字段，无数据时隐藏词语区域 |
| 小米字格布局适配 | 使用 flex 布局和 rpx 单位确保响应式 |
