# 汉字认字量科学检测 · 使用说明

> Skill 名称：`chinese-literacy-detection`
> 适用年龄：**3 - 12 岁**儿童

---

## 📌 功能介绍

本 Skill 通过**对话交互**，科学检测儿童汉字认字量。核心特点：

| 特性 | 说明 |
|------|------|
| **科学抽样** | 基于 2500 高频汉字（覆盖 98.5% 日常语料），6 层分层抽样，最多测 175 字即出结果 |
| **动态熔断** | 连续 5 字不认识或错误率 ≥ 80% 时自动停止，保护孩子心理感受 |
| **加权估算** | 不同层级按使用频率赋权，公式 `W = Σ(Nᵢ × Wᵢ)`，精度 ±100-150 字 |
| **正向鼓励** | 结果报告始终使用积极语言，从不出现负面评价 |
| **组词辅助** | 每个测试汉字配 2 个常见词组，帮助孩子联想回忆 |

### 适用场景

- 识字量测评 / 认字量检测
- 汉字认读水平评估
- 测试后生字复习与组词练习
- 了解检测的科学原理
- 同龄段识字水平参考对照

---

## 🚀 使用步骤

### 1️⃣ 触发技能

在对话中使用以下任意表达即可启动：

```
"测一下认字量"  "识字测试"  "帮孩子看看识字量"
"孩子认识多少字"  "识字水平怎么样"  "识字量达标了吗"
```

> 💡 也支持英文触发，如 `"Chinese character recognition test"`

### 2️⃣ 查看引导提示

启动后会展示**微信小程序二维码**——如需更流畅的触屏交互体验（孩子可直接点选），可扫码进入小程序；也可继续在当前对话中测试。

### 3️⃣ 输入年龄

告知被测儿童年龄（3-12 岁），系统据此提供同龄段参考。

### 4️⃣ 分批测试

系统按 L1（核心字）→ L6（拓展字）由易到难分批出题，每批约 10 字，以**三列表格**形式呈现：

| 序号 | 汉字 | 组词 |
|:----:|:----:|------|
| 1 | **的** | 好的、是的 |
| 2 | **一** | 一个、一样 |

**回复方式**（任选其一）：

| 回复示例 | 含义 |
|---------|------|
| `都认识` | 本批全部认识 |
| `都不认识` | 本批全部不认识 |
| `不认识：了、不` | 列出不认识的字 |
| `第3、5个不认识` | 按序号指定 |

### 5️⃣ 获取报告

测试结束（全部测完或触发熔断）后，自动生成结果报告，包含：

- **估算认字量**（加权总分）
- **各层级正确率**与估算详情
- **同龄段水平参考**与发展建议
- **不认识的字汇总**（按层级整理）

### 6️⃣ 后续互动（可选）

| 功能 | 说明 |
|------|------|
| 📚 生字复习 | 逐字展示不认识的汉字及组词，进行针对性学习 |
| 🔬 科学原理 | 了解分层抽样、加权估算和动态熔断的数学原理 |
| 📊 年龄参考 | 查看同龄段平均识字水平与发展阶段 |

## 📋 系统要求

| 项目 | 要求 |
|------|------|
| 运行环境 | 支持 Markdown 渲染的对话界面（如 CodeBuddy 等 AI 助手） |
| 字体显示 | 二维码引导需**等宽字体**环境才能正确显示 |
| 数据依赖 | Skill 内置 `assets/top_2500_chars_with_words.json` 数据资产 |
| 测试时长 | 约 **3-5 分钟**（视孩子水平，熔断后可能更短） |

---

## ⚠️ 注意事项

1. **年龄范围**：测试针对 3-12 岁设计，超出范围结果可能不准确
2. **辅助建议**：低龄儿童（3-5 岁）建议家长在旁协助指读和回复
3. **组词作用**：组词是帮助联想回忆的线索，不等于"会读组词就算认识"
4. **正向引导**：测试中若触发熔断，属正常现象，说明该难度段需要时间积累
5. **精度说明**：估算结果精度约 ±100-150 字，作为参考而非绝对标准
6. **中途退出**：如需中途结束，系统可基于已测部分给出参考结果（可能偏低）
7. **小程序体验**：对话式测试需手动打字回复，低龄儿童推荐使用小程序的触屏点选模式
