# Chatbot 对话式测试交互规范

## 1. 触发识别

### 1.1 触发关键词

以下用户输入应触发本 Skill：

| 类别 | 示例表达 |
|------|---------|
| 直接请求 | "测一下认字量""识字测试""检测认识多少字""开始测试" |
| 间接请求 | "孩子认字水平怎么样""7岁应该认识多少字""帮孩子看看识字量" |
| 原理询问 | "怎么测的""这个准吗""分层抽样是什么""熔断什么意思" |
| 复习请求 | "不认识的字有哪些""帮我复习生字""组词练习" |

### 1.2 流程选择

```
用户意图判断
  ├── 要求测试 → 走"完整测试流程"
  ├── 询问原理 → 走"科学原理说明流程"
  ├── 要求复习 → 走"生字复习流程"（需有历史记录）
  └── 查询参考 → 走"年龄参考查询流程"
```

## 2. 完整测试流程

### 2.1 Phase 1: 信息收集

```
Chatbot: 好的！我来帮你做一个科学的识字量检测。
        请问被测小朋友几岁了？（1-15岁）

User:   7岁

Chatbot: 7岁，正处于二年级水平的系统学习期！
        
        📋 测试说明：
        • 我会从常用 2500 字中抽取最多 175 个字
        • 分 6 个难度层级，由易到难
        • 我会分批给出汉字，请告诉我哪些不认识
        • 如果某层级连续不会，我会自动跳过避免挫败感
        • 全程约 3-5 分钟
        
        准备好了就说"开始"吧！
```

### 2.2 Phase 2: 即时生成测试序列 & 分层测试

#### 测试序列生成

在出题前，Chatbot 需要先即时生成测试序列：

1. **读取数据源**：`assets/top_2500_chars_with_words.json`（2500 条记录）
2. **分层过滤**：按 `rank_id` 将汉字分配到 6 个层级
3. **层内随机打乱**：每层内用 Fisher-Yates 洗牌随机排列
4. **抽样**：L1 全量（50字）、L2-L6 各按 testCount 取前 N 个
5. **拼接**：按 L1→L6 顺序拼接，得到 175 字的测试序列

> 也可执行 `scripts/generate_test_sequence.py` 生成序列

#### 出题策略

**批量模式**（推荐，每批 5-10 字）：

```
Chatbot: 📝 【L1·核心字】第 1 组（1-10 / 共 50 字）
        
        的、一、是、了、不、在、人、有、我、他
        
        👆 上面哪些字不认识？都认识就回复"都认识"

User:   都认识

Chatbot: ✅ 全部正确！继续下一组～
        
        📝 【L1·核心字】第 2 组（11-20 / 共 50 字）
        
        这、中、大、来、上、国、个、到、说、们
        
        不认识的字有哪些？

User:   "个"不太确定

Chatbot: 好的，"个"标记为不认识。
        📝 组词参考：个人、个子、一个
        
        继续！
```

**逐字模式**（适合低龄 / 精细测试）：

```
Chatbot: 🔤 第 1 个字（1/175）【L1·核心字】
        
           的
        
        组词：的确、目的、的当
        
        认识吗？（认识 / 不认识）

User:   认识

Chatbot: ✅ 下一个！
        
        🔤 第 2 个字（2/175）【L1·核心字】
        
           一
        
        组词：一起、一个、一样
        
        认识吗？
```

#### ⚠️ 出题数量强制约束

每层出题总数必须严格等于 `LEVEL_CONFIGS` 中定义的 `testCount`：

| 层级 | testCount | 说明 |
|------|:---------:|------|
| L1 | **50** | 全量测试 |
| L2 | **50** | 从150字中抽50 |
| L3 | **30** | 从300字中抽30 |
| L4 | **25** | 从500字中抽25 |
| L5 | **10** | 从500字中抽10 |
| L6 | **10** | 从1000字中抽10 |

- 每批出题数 = `min(10, 该层剩余未出题数)`
- 当某层已出题数 = testCount 时，必须切换到下一层
- **禁止多出或少出**

#### ⚠️ Chatbot 内部状态追踪（每批回复后必须更新）

Chatbot 在每批处理后，**必须**内部维护以下状态（无需展示给用户，但必须在内部严格追踪）：

```
当前层级: L{level}
该层 testCount: {N}
该层已测: {tested}
该层剩余: {remaining}
该层认识: {known}
该层不认识: {unknown}
该层 consecutiveUnknown: {count}
该层 errorRate: {unknown}/{tested} = {rate}%
→ 熔断判定: {是/否} (原因: ...)
→ 下一步: {继续出题 / 触发熔断跳转结果报告}
```

**此状态检查是每批回复后的必经步骤，不得省略。**

#### ⚠️ 强制熔断检查流程（每批回复后必须执行）

每收到用户回复后，**必须**按以下顺序执行：

1. **按出题顺序逐字处理**：将用户回复映射到出题顺序，逐字更新状态
   - 认识 → `consecutiveUnknown = 0`
   - 不认识 → `consecutiveUnknown += 1`
   - **如果处理过程中 `consecutiveUnknown` 达到 5，立即停止处理，触发熔断**

2. **检查熔断条件**：
   ```
   IF consecutiveUnknown ≥ 5 → 触发熔断
   IF 已测 ≥ 5 AND 不认识/已测 ≥ 0.8 → 触发熔断
   ```

3. **"都不认识"快速路径**：
   - 用户回复"都不认识" → 本批全部标记 unknown
   - `consecutiveUnknown += 本批字数`（必然 ≥ 5）
   - **必须触发熔断，禁止继续出题**

#### 层级过渡提示

```
Chatbot: ✅ L1·核心字 测试完成！ 48/50 正确 (96%)
        
        ⏩ 进入下一层级：L2·常用字（50字）
        难度略有提升，别紧张，继续加油！
        
        📝 【L2·常用字】第 1 组（1-10 / 共 50 字）
        ...
```

### 2.3 Phase 3: 熔断处理

当触发熔断条件时：

**连续失败熔断**：

```
Chatbot: ⚡ 智能检测提示
        
        当前层级（L4·进阶字）已经连续 5 个字不认识了。
        这完全正常——说明这个难度段的字还需要时间积累。
        
        为了不让孩子感到压力，后续更难的层级我就不测了，
        直接给你看结果！
```

**错误率熔断**：

```
Chatbot: ⚡ 智能检测提示
        
        当前层级（L3·扩展字）已测 8 字，其中 7 字不认识（错误率 87.5%）。
        说明这个难度段对孩子来说还比较超前，这很正常！
        
        跳过后续层级，直接计算结果～
```

**"都不认识"触发熔断**：

```
User:   都不认识

Chatbot: ⚡ 智能检测提示
        
        当前层级（L4·进阶字）本批 10 个字全部不认识，
        连续不认识数已达到 10 个，说明这个难度段还需要积累。
        
        为了不让孩子感到压力，后续更难的层级我就不测了，
        直接给你看结果！
```

**批量模式中的连续判断示例**：

```
出题顺序：海、山、西、美、口、角、走、干、特、员
用户回复："海、山、西、美、口不认识，其他都认识"

处理过程（按出题顺序）：
  海(不认识) → consecutive=1
  山(不认识) → consecutive=2
  西(不认识) → consecutive=3
  美(不认识) → consecutive=4
  口(不认识) → consecutive=5 ⚡ 触发熔断！
  → 后续字（角、走、干、特、员）不再处理
  → 立即跳转结果报告
```

> ⚠️ **关键规则**：即使不认识的字散布在一批中（如"第1、3、5、7、9个不认识"），
> 也必须按出题顺序逐字更新 `consecutiveUnknown`。如果中间有认识的字，`consecutiveUnknown` 重置为 0。

## 3. 科学原理说明流程

当用户询问测试原理时的输出模板：

```
Chatbot: ## 🔬 识字量检测的科学原理
        
        ### 1️⃣ 数据来源
        我们使用的字表来自**现代汉语大规模语料库统计**，
        选取使用频率最高的 2500 个汉字。这 2500 字覆盖了
        日常阅读材料中 **98.5%** 的内容。
        换句话说，认识这 2500 字就能无障碍阅读几乎所有中文文本。
        
        ### 2️⃣ 分层抽样策略
        我们不会傻傻地测全部 2500 个字（那太累了！），
        而是把它们按使用频率分成 6 层：
        
        • L1（前50字）：全部测试——这些是"的、一、是、了"这样的超高频字
        • L2-L6：按比例抽样——越低频的层，抽样越稀疏
        
        这样最多只需测 **175 个字**，就能推算完整认字量。
        
        ### 3️⃣ 计算方法
        核心公式：**认字量 = Σ(每层认识数 × 权重)**
        
        举例：如果在 L3 层（共 300 字）中抽了 30 字，认识了 20 字，
        那就估算 L3 层认识 20 × 10 = 200 字。
        6 层加起来就是总认字量。
        
        ### 4️⃣ 动态熔断
        如果在某层连续 5 个字不认识，或错误率超过 80%，
        就自动停止——因为更难的字大概率也不会认识了。
        这既节省时间，也避免孩子产生挫败感。
        
        ### 5️⃣ 精度
        这种方法的 95% 置信区间大约 **±100-150 字**。
        足以判断发展阶段和大致水平。
        多次测试取平均可以更精确。
```

基于测试结果中的不认识的字进行学习：

```
Chatbot: 📚 开始复习不认识的字！共 37 个字，我一个一个来。
        
        第 1 个：
        
        ## 凡
        
        📝 组词：凡是、平凡、非凡
        🔢 词频排名：#42（属于最常用的 50 字之一）
        
        看一看，记住了吗？准备好了回复"下一个"！

User:   下一个

Chatbot: 第 2 个：
        
        ## 乃
        
        📝 组词：乃至、乃是
        🔢 词频排名：#38
        
        这个字在古文中很常见哦！"乃"就是"是"的意思。
        记住了吗？
```

## 5. 输出格式规范

### 5.1 汉字展示格式

在对话中展示汉字时：

- **批量出题**：用顿号分隔，如 `的、一、是、了、不`
- **单字展示**：用大号标题格式，如 `## 的`
- **附带组词**：`📝 组词：的确、目的、的当`
- **附带排名**：`🔢 排名：#1 (L1·核心字)`

### 5.2 进度显示格式

```
📝 【L{level}·{name}】第 {batch} 组（{start}-{end} / 共 {total} 字）
```

### 5.3 层级完成格式

```
✅ L{level}·{name} 测试完成！{known}/{tested} 正确 ({rate}%)
```

### 5.4 Emoji 使用规范

| Emoji | 用途 |
|-------|------|
| 📝 | 出题、测试相关 |
| ✅ | 正确、完成 |
| ❌ | 不认识 |
| ⚡ | 熔断提示 |
| 📊 | 结果报告 |
| 🎯 | 核心数据（认字量） |
| 💪 | 鼓励评语 |
| 📖 | 生字列表 |
| 📚 | 学习/复习 |
| 🔬 | 科学原理 |
| 🔢 | 排名/数字 |
| 💡 | 建议/提示 |
| ⏩ | 层级过渡 |

## 6. 用户回复解析

### 6.1 批量模式回复解析

用户可能的回复方式及解析规则：

| 用户回复 | 解析 |
|---------|------|
| "都认识" / "全认识" / "没有不认识的" | 本批全部标记为 known |
| "都不认识" / "一个都不会" | 本批全部标记为 unknown |
| "不认识：了、不" / "'了'和'不'" | 指定的字标记为 unknown，其余 known |
| "认识前5个" / "后面3个不认识" | 按位置标记 |
| "第3个第5个不认识" | 按序号标记 |

### 6.2 逐字模式回复解析

| 用户回复 | 解析 |
|---------|------|
| "认识" / "会" / "知道" / "对" / "嗯" | known |
| "不认识" / "不会" / "不知道" / "没见过" | unknown |
| "不确定" / "好像见过" | 建议标记为 unknown |

### 6.3 模糊回复处理

```
User:   不太确定

Chatbot: 没关系！不确定的话我们先标记为"不认识"，
        这样结果会更保守、更准确。
        后续可以再学习这个字哦！
```

## 7. 异常处理

### 7.1 用户中途退出

```
User:   不想测了 / 太累了

Chatbot: 没问题！测试可以随时暂停。
        
        目前已完成的部分也可以给你一个参考结果：
        [基于已测部分计算并展示结果]
        
        不过因为没有测完所有层级，这个结果可能偏低。
        想继续测的时候随时告诉我！
```

### 7.2 年龄超出范围

```
User:   16岁

Chatbot: 这个测试主要针对 1-15 岁的儿童设计。
        16 岁通常已经掌握了 3000+ 汉字，超出了本测试的 2500 字范围。
        
        如果还是想测一下基础认字量，我可以继续。
        不过结果可能无法准确反映真实水平哦。
        要继续吗？
```

### 7.3 回复无法解析

```
User:   😊

Chatbot: 抱歉我没太理解，请告诉我上面的字里哪些**不认识**？
        
        你可以这样回复：
        • "都认识" — 全部认识
        • "不认识：X、Y" — 列出不认识的字
        • "都不认识" — 全部不认识
```
