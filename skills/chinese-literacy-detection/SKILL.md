# 汉字认字量科学检测 Chatbot Skill

## 概述

本 Skill 指导 Chatbot 通过**对话交互**完成儿童（1-15岁）汉字认字量的科学检测。基于**高频汉字分层抽样 + 动态熔断**方法，从 2500 个高频汉字（覆盖日常语料 98.5%）中，最多测试 **175 字**即可精准估算认字量。

**核心能力**：对话即测试——无需 UI 界面，Chatbot 逐字/批量出题，用户文字回答，实时计算并反馈结果。

## 适用场景

当用户在对话中提出以下意图时触发本 Skill：

- 「帮我/帮孩子测一下认字量」「识字测试」「检测认识多少字」
- 「我家孩子 X 岁，认字水平怎么样？」
- 「这个识字测试的原理是什么？」「怎么计算的？」
- 「给我解释一下分层抽样 / 熔断机制」
- 「帮孩子练习不认识的字」「组词练习」

## 核心算法：分层抽样测试

### 六层分级配置

```
层级  名称      排名区间       测试数  权重   说明
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
L1   核心字    1-50           50     ×1    最高频50字，全部测试
L2   常用字    51-200         50     ×3    150字中随机抽50
L3   扩展字    201-500        30     ×10   300字中随机抽30
L4   进阶字    501-1000       25     ×20   500字中随机抽25
L5   提高字    1001-1500      10     ×50   500字中随机抽10
L6   拓展字    1501-2500      10     ×100  1000字中随机抽10
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
总计                          175
```

### 核心配置数据

```javascript
const LEVEL_CONFIGS = [
  { level: 1, name: '核心字', rankStart: 1,    rankEnd: 50,   sampleInterval: 1,   testCount: 50,  weight: 1   },
  { level: 2, name: '常用字', rankStart: 51,   rankEnd: 200,  sampleInterval: 3,   testCount: 50,  weight: 3   },
  { level: 3, name: '扩展字', rankStart: 201,  rankEnd: 500,  sampleInterval: 10,  testCount: 30,  weight: 10  },
  { level: 4, name: '进阶字', rankStart: 501,  rankEnd: 1000, sampleInterval: 20,  testCount: 25,  weight: 20  },
  { level: 5, name: '提高字', rankStart: 1001, rankEnd: 1500, sampleInterval: 50,  testCount: 10,  weight: 50  },
  { level: 6, name: '拓展字', rankStart: 1501, rankEnd: 2500, sampleInterval: 100, testCount: 10,  weight: 100 },
];

const FUSE_CONFIG = {
  consecutiveUnknownLimit: 5,  // 连续不认识触发熔断
  errorRateLimit: 0.8,         // 错误率触发熔断
  minTestCountForErrorRate: 5, // 错误率计算最少样本
};
```

> ⚠️ **强制执行规则**：
> 1. **出题数量**：每层出题总数必须严格等于该层 `testCount`（L1:50, L2:50, L3:30, L4:25, L5:10, L6:10，总计 175），不得多出或少出。
> 2. **熔断检查**：每收到一次用户回复，必须立即按 `FUSE_CONFIG` 执行熔断检查。若触发熔断，**禁止继续出题**，必须立即跳转到结果报告。
> 3. **"都不认识"场景**：用户回复"都不认识"时，本批所有字标记为 unknown，`consecutiveUnknown` 必然 ≥ 5（每批至少5字），**必须触发熔断**。

### 认字量计算公式

```
估算认字量 W = Σ (层级认识数 Nᵢ × 权重 Wᵢ)
             = N₁×1 + N₂×3 + N₃×10 + N₄×20 + N₅×50 + N₆×100
```

**计算示例**：L1: 48/50, L2: 40/50, L3: 20/30, L4: 15/25, L5: 5/10, L6: 2/10

```
W = 48×1 + 40×3 + 20×10 + 15×20 + 5×50 + 2×100
  = 48 + 120 + 200 + 300 + 250 + 200 = 1118 字
```

### 动态熔断机制

在测试过程中的自适应停止策略，避免对儿童造成挫败感：

**触发条件**（任一满足即熔断）：
1. **连续失败熔断**：当前层级连续 5 个字不认识
2. **错误率熔断**：当前层级已测 ≥ 5 字且错误率 > 80%

**熔断后处理**：
- 当前层级剩余未测字 → 全部视为"不认识"
- 更高层级全部跳过 → 所有字视为"不认识"
- 立即计算结果并反馈

## Chatbot 对话测试流程

### 流程概览

```
用户触发 → 询问年龄 → 简要说明 → 逐批出题(L1→L6) → 实时熔断检查 → 计算结果 → 报告反馈
```

### Step 1: 测试启动

用户表达测试意图后，Chatbot 需要收集：
- **必填**：被测儿童的年龄（1-15岁）
- **可选**：是否之前测试过

### Step 2: 测试说明

向用户（通常是家长）简要说明：

> 好的！我将通过分层抽样的科学方法，从常用 2500 个汉字中抽取最多 175 个字来测试。
> 
> 我会分批给出汉字，请让孩子告诉我哪些字认识、哪些不认识。
> 测试分 6 个难度层级，由易到难。如果某个层级连续多个字不认识，我会自动停止避免挫败感。
> 
> 准备好了吗？我们开始吧！

### Step 3: 逐批出题

#### ⚠️ 出题数量强制约束（必须严格遵守）

**每层出题前，必须查阅 LEVEL_CONFIGS 确认该层 testCount，并严格按此数量出题：**

| 层级 | testCount（严格遵守） | 每批出题数 |
|------|:---:|:---:|
| L1 核心字 | **50** | min(10, 剩余) |
| L2 常用字 | **50** | min(10, 剩余) |
| L3 扩展字 | **30** | min(10, 剩余) |
| L4 进阶字 | **25** | min(10, 剩余) |
| L5 提高字 | **10** | min(10, 剩余) |
| L6 拓展字 | **10** | min(10, 剩余) |

- 每层的出题总数必须 **严格等于** 该层 `testCount`，不得多出或少出
- 每批出题数 = `min(10, 该层剩余未出题数)`
- 当某层出题数已达 `testCount` 时，必须切换到下一层
- **禁止凭感觉分配出题数量，必须精确追踪每层已出题数**

**推荐批量模式**（效率高，适合 7 岁以上）：

每次出 5-10 个字，让用户标注不认识的字：

> 【第 1 层·核心字】第 1-10 个（共 50 个）
> 
> **的、一、是、了、不、在、人、有、我、他**
> 
> 请告诉我上面哪些字**不认识**？（都认识就回复"都认识"）

**逐字模式**（适合低龄 / 用户偏好）：

> 🔤 第 1 个字（共 175 个）【核心字·L1】
> 
> # 的
> 
> 组词：的确、目的、的当
> 
> 认识这个字吗？

### Step 4: 记录与熔断检查（⚠️ 每批回复后必须执行，不得跳过）

**每收到用户回复后，必须严格按以下顺序执行，不得省略任何步骤：**

#### 4.1 按出题顺序逐字更新状态

将用户回复映射到出题顺序，**按出题顺序逐字**更新以下状态：
- 如果该字"认识"：`consecutiveUnknown = 0`（重置）
- 如果该字"不认识"：`consecutiveUnknown += 1`

**⚠️ 批量模式关键规则**：在批量模式下，必须按照**出题顺序**逐字处理用户回复来维护 `consecutiveUnknown`。

示例：出题顺序为 A B C D E F G H I J，用户说"A B C D E 不认识，其他都认识"
→ 处理顺序：A(不认识, consecutive=1) → B(不认识, consecutive=2) → C(不认识, consecutive=3) → D(不认识, consecutive=4) → E(不认识, consecutive=5) → **此时 consecutive=5，立即触发熔断！F-J 不再处理。**

#### 4.2 强制熔断检查（⚠️ 每批回复后必须执行此判断）

```
// 条件 A：连续失败熔断
IF consecutiveUnknown ≥ 5:
  → 立即触发熔断，停止出题，跳转到 Step 5 结果报告

// 条件 B：错误率熔断
IF 该层已测字数 ≥ 5 AND 不认识数 / 已测数 ≥ 0.8:
  → 立即触发熔断，停止出题，跳转到 Step 5 结果报告
```

**判断完毕后才能决定下一步**：
- 未熔断 → 继续下一批出题
- 已熔断 → 通知用户并跳到 Step 5 结果报告

#### 4.3 "都不认识"快速路径

当用户回复"都不认识"时：
1. 本批**所有字**标记为 unknown
2. `consecutiveUnknown += 本批字数`
3. 由于每批至少 5 个字，`consecutiveUnknown` **必然 ≥ 5** → **必须触发熔断**
4. **禁止继续出下一批题**，必须立即跳转到 Step 5 结果报告

> ⚠️ **强制规则**：Chatbot 在测试过程中，每收到一次用户回复，必须立即按 `FUSE_CONFIG` 检查熔断条件。若触发熔断，**禁止继续出下一批题**，必须立即跳转到结果报告。

**熔断通知示例**：

> ⚡ 检测到当前层级（L3·扩展字）连续 5 个字不认识，为避免孩子产生挫败感，自动跳过后续更难的层级。
> 
> 这不代表水平不好——只是说明这个难度段还需要积累，接下来我给你看看完整结果！

### Step 5: 结果计算与报告

测试结束后（全部测完或熔断），按公式计算并输出结构化报告：

> ## 📊 测试结果报告
> 
> 👤 被测儿童：7 岁
> 📝 实际测试：135 个字
> ✅ 认识：98 个字
> ❌ 不认识：37 个字
> 📏 正确率：72.6%
> ⚡ 熔断：L5·提高字（连续5个字不认识）
> 
> ### 🎯 估算认字量：**1,118 字**
> 
> ### 各层级详情
> | 层级 | 认识/测试 | 正确率 | 估算认识 |
> |------|-----------|--------|----------|
> | L1 核心字 | 48/50 | 96% | 48 字 |
> | L2 常用字 | 40/50 | 80% | 120 字 |
> | L3 扩展字 | 20/30 | 67% | 200 字 |
> | L4 进阶字 | 15/25 | 60% | 300 字 |
> | L5 提高字 | 5/10 | 50% | 250 字 ⚡ |
> | L6 拓展字 | — | — | 200 字 |
> 
> ### 💪 评估
> 7 岁（二年级水平）预期认字量为 1200-1600 字。
> 1,118 字接近预期下限，属于正常发展范围，继续保持阅读习惯即可稳步提升！

## 科学原理说明

当用户询问测试原理时，按以下框架解释：

### 1. 数据来源
基于现代汉语大规模语料库统计，前 2500 高频字覆盖日常阅读 98.5% 的内容。

### 2. 分层抽样
将 2500 字按词频分为 6 层，高频层全测，低频层抽样。核心原理：同一频率段的字被认识的概率相近（层内同质性），用少量样本可推算整层。

### 3. 加权估算
每层用样本认识率 × 层级总字数得到估算值，6 层求和即总认字量。公式：`W = Σ(Nᵢ × Wᵢ)`

### 4. 动态熔断
连续不认识 5 字或错误率 > 80% 时自动停止，保护儿童心理，同时节省测试时间。

### 5. 精度说明
95% 置信区间约 ±100-150 字。多次测试取平均可进一步提高精度。

根据认字量和年龄生成正向鼓励语：

## 测试记录数据结构

每次测试完成后应生成如下结构化记录：

```typescript
interface TestRecord {
  id: string;                    // 唯一标识
  timestamp: number;             // 时间戳
  totalVocabulary: number;       // 估算认字量
  totalTested: number;           // 实际测试字数
  totalKnown: number;            // 认识的字数
  totalUnknown: number;          // 不认识的字数
  accuracy: number;              // 正确率 (0-1)
  fuseTriggered: boolean;        // 是否触发熔断
  fuseLevel: number | null;      // 熔断层级
  fuseReason: string;            // 熔断原因
  levelResults: LevelResult[];   // 各层级结果
  unknownChars: UnknownChar[];   // 不认识的字
  userAge: number;               // 被测儿童年龄
}

interface LevelResult {
  level: number;              // 层级 (1-6)
  name: string;               // 层级名称
  tested: number;             // 测试字数
  known: number;              // 认识数
  unknown: number;            // 不认识数
  estimatedKnown: number;     // 估算认识数 = known × weight
  weight: number;             // 权重
  fuseTriggered: boolean;     // 是否在该层熔断
}

interface UnknownChar {
  char: string;               // 汉字
  rank_id: number;            // 排名
  level: number;              // 层级
  words: string[];            // 组词
}
```
