# Chatbot 对话式测试交互规范

## 1. 触发识别

### 1.1 触发关键词

| 类别 | 示例表达 |
|------|---------|
| 直接请求 | "测一下认字量""识字测试""检测认识多少字""开始测试" |
| 间接请求 | "孩子认字水平怎么样""7岁应该认识多少字""帮孩子看看识字量" |
| 原理询问 | "怎么测的""这个准吗""分层抽样是什么""熔断什么意思" |
| 复习请求 | "不认识的字有哪些""帮我复习生字""组词练习" |

### 1.2 流程选择

```
用户意图判断
  ├── 要求测试 → 走"完整测试流程"
  ├── 询问原理 → 走"科学原理说明流程"
  ├── 要求复习 → 走"生字复习流程"（需有历史记录）
  └── 查询参考 → 走"年龄参考查询流程"
```

## 2. 完整测试流程

### 2.1 Phase 1: 信息收集

```
Chatbot: 好的！我来帮你做一个科学的识字量检测。
        请问被测小朋友几岁了？（1-15岁）

User:   7岁

Chatbot: 7岁，正处于二年级水平的系统学习期！
        
        📋 测试说明：
        • 我会从常用 2500 字中抽取最多 175 个字
        • 分 6 个难度层级，由易到难
        • 我会分批给出汉字，请告诉我哪些不认识
        • 如果某层级连续不会，我会自动跳过避免挫败感
        • 全程约 3-5 分钟
        
        准备好了就说"开始"吧！
```

### 2.2 Phase 2: 即时生成测试序列 & 分层测试

#### 测试序列生成

1. 读取 `assets/top_2500_chars_with_words.json`
2. 按 `rank_id` 分配到 6 层
3. 层内 Fisher-Yates 洗牌随机排列
4. L1 全量（50字），L2-L6 各按 testCount 取前 N 个
5. 按 L1→L6 拼接得到 175 字测试序列

#### 出题策略

> **🚨 严格约定：每一组汉字必须且只能以「三列表格（序号｜汉字｜组词）」形式展示。此规则与熔断规则同等优先级，不得违反。**

**唯一合法的出题格式**（每批 min(10, 剩余) 字，表格展示含组词）：

```
Chatbot: 📝 【L1·核心字】第 1 组（1-10 / 共 50 字）
        
        | 序号 | 汉字 | 组词 |
        |:----:|:----:|------|
        | 1 | **的** | 好的、是的 |
        | 2 | **一** | 一个、一样 |
        | 3 | **是** | 但是、总是 |
        | 4 | **了** | 了解、去了 |
        | 5 | **不** | 不要、不去 |
        | 6 | **在** | 在家、存在 |
        | 7 | **人** | 人们、大人 |
        | 8 | **有** | 有趣、没有 |
        | 9 | **我** | 我们、自我 |
        | 10 | **他** | 他们、其他 |
        
        👆 上面哪些字不认识？都认识就回复"都认识"
```

**第 2 组序号延续示例**（严格执行跨组连续编号！）：

```
Chatbot: 📝 【L1·核心字】第 2 组（11-20 / 共 50 字）
        
        | 序号 | 汉字 | 组词 |
        |:----:|:----:|------|
        | 11 | **这** | 这里、这个 |
        | 12 | **上** | 上面、上课 |
        | 13 | **大** | 大家、大树 |
        | 14 | **来** | 回来、过来 |
        | 15 | **和** | 和平、和好 |
        | 16 | **个** | 个人、这个 |
        | 17 | **中** | 中间、中国 |
        | 18 | **地** | 地方、地图 |
        | 19 | **为** | 因为、为了 |
        | 20 | **生** | 生活、生命 |
        
        👆 上面哪些字不认识？都认识就回复"都认识"
```

**表格格式强制要求**：

| 要求 | 说明 |
|------|------|
| 必须三列 | 「序号」「汉字」「组词」三列缺一不可 |
| 组词不得为空 | 每行组词列必须至少 1 个词组，来自数据源 `words` 字段 |
| 汉字必须加粗 | 使用 `**字**` 格式，方便孩子辨认 |
| 序号层级内全局连续 | 同一层级内跨组连续递增，不得每组重新从 1 开始。第 1 组 1-10，第 2 组 11-20，以此类推。用户回复"第N个不认识"时 N 为全局序号 |
| 组词分隔符 | 多个词组用顿号「、」分隔 |

**禁止出现的格式（违反即为 Bug）**：
- ❌ `的 一 是 了 不`（纯文本排列，无表格，无组词）
- ❌ `# 的　一　是　了　不`（大字展示，无表格，无组词）
- ❌ 只有「序号｜汉字」两列而缺少「组词」列的表格
- ❌ 组词列为空或显示"—"的表格行
- ❌ 任何非 Markdown 表格的出题形式（包括列表、逗号分隔、空格排列等）

**逐字模式**（适合低龄 / 用户偏好，同样必须包含组词）：

```
Chatbot: 🔤 第 1 个字（1/175）【L1·核心字】
        
        | 汉字 | 组词 |
        |:----:|------|
        | **的** | 的确、目的、的当 |
        
        认识吗？（认识 / 不认识）
```

> 逐字模式也必须以表格形式展示，包含「汉字」和「组词」两列，不得省略组词。

#### 出题数量强制约束

| 层级 | testCount | 每批出题数 |
|------|:---------:|:----------:|
| L1 | **50** | min(10, 剩余) |
| L2 | **50** | min(10, 剩余) |
| L3 | **30** | min(10, 剩余) |
| L4 | **25** | min(10, 剩余) |
| L5 | **10** | min(10, 剩余) |
| L6 | **10** | min(10, 剩余) |

- 每层出题数 **严格等于** testCount，不得多出或少出
- 当某层已出题数 = testCount 时，必须切换到下一层

#### 层级过渡提示

```
Chatbot: ✅ L1·核心字 测试完成！ 48/50 正确 (96%)
        
        ⏩ 进入下一层级：L2·常用字（50字）
        难度略有提升，别紧张，继续加油！
        （注意：新层级序号从 1 重新开始）
```

## 3. ⛔ 熔断检查执行规范（每批回复后强制执行）

> **🚨 这是整个 Skill 的最高优先级规则。每收到一次用户回复，必须先完成本节全部步骤，才能决定下一步。**

### 3.1 强制状态追踪输出

收到用户回复后，**必须先在回复中输出以下状态块**（不可省略），再决定是否继续出题：

```
📋 状态追踪：
- 当前层级：L{level}·{name}
- 本批出题：序号 {globalStart}-{globalEnd}，{N} 字（{字1}、{字2}、...）
- 用户回复：{解析结果}（注意：用户回复的序号为层级内全局序号）
- 逐字处理：{字1}({认识/不认识}, consecutive={N}) → {字2}(...) → ...
- consecutiveUnknown：{之前值} → {当前值}
- 该层已测/testCount：{tested}/{testCount}
- 该层不认识数：{unknown}
- 🔍 熔断检查 A：consecutiveUnknown({value}) ≥ 5 → {是/否}
- 🔍 熔断检查 B：已测({tested}) ≥ 5 且 错误率({unknown}/{tested}={rate}%) ≥ 80% → {是/否}
- ✅ 结论：{继续出题 / 🔴 触发熔断}
```

**示例——用户回复"都不认识"**：

```
📋 状态追踪：
- 当前层级：L1·核心字
- 本批出题：序号 1-10，10 字（的、一、是、了、不、在、人、有、我、他）
- 用户回复：都不认识 → 10 字全部标记为 unknown
- 逐字处理：的(✗,1) → 一(✗,2) → 是(✗,3) → 了(✗,4) → 不(✗,5) → ⚡到达阈值5，停止处理！
- consecutiveUnknown：0 → 10（+10）
- 该层已测/testCount：10/50
- 该层不认识数：10
- 🔍 熔断检查 A：consecutiveUnknown(10) ≥ 5 → ✅ 触发！
- 🔍 熔断检查 B：已测(10) ≥ 5 且 错误率(10/10=100%) ≥ 80% → ✅ 触发！
- ✅ 结论：🔴 触发熔断！立即跳转结果报告。
```

**示例——第2组用户回复"第13、15个不认识"（出题序号 11-20，对应 A B C D E F G H I J）**：

```
📋 状态追踪：
- 当前层级：L2·常用字
- 本批出题：序号 11-20，10 字（A、B、C、D、E、F、G、H、I、J）
- 用户回复：第13(C)、第15(E) 不认识，其余认识
- 逐字处理：A(✓,0) → B(✓,0) → C(✗,1) → D(✓,0) → E(✗,1) → F(✓,0) → G(✓,0) → H(✓,0) → I(✓,0) → J(✓,0)
- consecutiveUnknown：0 → 0
- 该层已测/testCount：20/50
- 该层不认识数：4
- 🔍 熔断检查 A：consecutiveUnknown(0) ≥ 5 → ❌ 未触发
- 🔍 熔断检查 B：已测(20) ≥ 5 且 错误率(4/20=20%) ≥ 80% → ❌ 未触发
- ✅ 结论：继续出题
```

### 3.2 逐字更新规则

按**出题顺序**逐字处理用户回复：
- 认识 → `consecutiveUnknown = 0`（重置）
- 不认识 → `consecutiveUnknown += 1`
- **如果处理过程中 `consecutiveUnknown` 达到 5，立即停止处理剩余字，触发熔断**

### 3.3 熔断条件

```
条件 A：IF consecutiveUnknown ≥ 5 → 🔴 立即熔断
条件 B：IF 已测 ≥ 5 AND 不认识/已测 ≥ 0.8 → 🔴 立即熔断
```

### 3.4 "都不认识"快速路径

用户回复"都不认识"时：
1. 本批**所有字**标记为 unknown
2. `consecutiveUnknown += 本批字数`（每批至少 5 字）
3. `consecutiveUnknown` **必然 ≥ 5** → **必须触发熔断**
4. **🚫 绝对禁令：下一条消息只能是熔断通知 + 结果报告，绝对不允许出现下一批题目**

### 3.5 出题前置门控

**每次生成下一批题目之前**，必须先检查：
- IF `consecutiveUnknown ≥ 5` → **禁止出题**
- IF 当前层已测 ≥ 5 且错误率 ≥ 80% → **禁止出题**

只有两个条件都不满足时，才允许生成下一批题目。

## 6. 用户回复解析

### 6.1 批量模式

| 用户回复 | 解析 |
|---------|------|
| "都认识" / "全认识" / "没有不认识的" | 本批全部 known |
| "都不认识" / "一个都不会" | 本批全部 unknown → **必须触发熔断** |
| "不认识：了、不" / "'了'和'不'" | 指定字 unknown，其余 known |
| "认识前5个" / "后面3个不认识" | 按位置标记（位置指本批内相对位置） |
| "第13个第15个不认识" | 按**层级内全局序号**定位对应字标记 unknown |

### 6.2 逐字模式

| 用户回复 | 解析 |
|---------|------|
| "认识" / "会" / "知道" / "嗯" | known |
| "不认识" / "不会" / "没见过" | unknown |
| "不确定" / "好像见过" | 标记为 unknown |

### 6.3 模糊回复

```
Chatbot: 没关系！不确定的话先标记为"不认识"，
        这样结果会更保守、更准确。后续可以再学习！
```

## 7. 异常处理

### 7.1 用户中途退出

基于已测部分计算并展示参考结果，提示因未测完可能偏低。

### 7.2 年龄超出范围

提示本测试针对 1-15 岁设计，超出范围结果可能不准确，征求用户意见是否继续。

### 7.3 回复无法解析

```
Chatbot: 请告诉我上面的字里哪些**不认识**？
        你可以这样回复：
        • "都认识" — 全部认识
        • "不认识：X、Y" — 列出不认识的字
        • "都不认识" — 全部不认识
```
