# 汉字认字量科学检测 Chatbot Skill

通过**对话交互**完成儿童（1-15岁）汉字认字量科学检测。基于**分层抽样 + 动态熔断**，从 2500 高频汉字中最多测 **175 字**即可精准估算认字量。

**数据源**：`assets/top_2500_chars_with_words.json`（2500 条，含 rank_id/char/words/frequency 字段）

**参考文档**（按需读取）：
- `references/algorithm-spec.md` — 算法数学规格、Fisher-Yates 洗牌、精度分析
- `references/data-schema.md` — 数据源字段说明与统计特征
- `references/chatbot-workflow.md` — 对话交互模板、输出格式、用户回复解析规则

## 核心配置

```javascript
const LEVEL_CONFIGS = [
  { level: 1, name: '核心字', rankStart: 1,    rankEnd: 50,   testCount: 50,  weight: 1   },
  { level: 2, name: '常用字', rankStart: 51,   rankEnd: 200,  testCount: 50,  weight: 3   },
  { level: 3, name: '扩展字', rankStart: 201,  rankEnd: 500,  testCount: 30,  weight: 10  },
  { level: 4, name: '进阶字', rankStart: 501,  rankEnd: 1000, testCount: 25,  weight: 20  },
  { level: 5, name: '提高字', rankStart: 1001, rankEnd: 1500, testCount: 10,  weight: 50  },
  { level: 6, name: '拓展字', rankStart: 1501, rankEnd: 2500, testCount: 10,  weight: 100 },
];

const FUSE_CONFIG = {
  consecutiveUnknownLimit: 5,  // 连续不认识触发熔断
  errorRateLimit: 0.8,         // 错误率触发熔断
  minTestCountForErrorRate: 5, // 错误率计算最少样本
};
```

**认字量公式**：`W = Σ(层级认识数 Nᵢ × 权重 Wᵢ)` = N₁×1 + N₂×3 + N₃×10 + N₄×20 + N₅×50 + N₆×100

**年龄参考**：3-4岁 50-200（启蒙期）| 4-5岁 200-500（兴趣培养期）| 5-6岁 500-800（入学准备期）| 6-7岁 800-1200（一年级）| 7-8岁 1200-1600（二年级）| 8-9岁 1600-2000（三年级）| 9-10岁 2000-2500（四年级）| 10岁+ 2500+

## 测试流程（必须严格按顺序执行）

### Step 1: 收集信息 → Step 2: 说明 → Step 3: 出题 → Step 4: 熔断检查 → Step 5: 报告

### Step 1 & 2: 启动

收集**年龄**（必填，1-15岁），简要说明测试方法后开始。

### Step 3: 逐批出题

从 `assets/top_2500_chars_with_words.json` 读取数据，按层级分层抽样生成测试序列。

**出题数量强制约束**：每层出题总数 **严格等于** 该层 `testCount`（L1:50, L2:50, L3:30, L4:25, L5:10, L6:10）。每批出题数 = `min(10, 该层剩余未出题数)`。

---

### ⚠️ 出题展示格式（严格约定，不得违反）

> **🚨 绝对规则：每一组汉字必须且只能以「表格 + 组词」形式展示。违反此规则视为严重 Bug，与熔断规则同等优先级。**

**强制要求**：
1. **必须使用 Markdown 表格**：每一组汉字展示时，**必须**使用包含「序号｜汉字｜组词」三列的 Markdown 表格，**禁止**使用纯文本列举、逗号分隔、空格排列等任何非表格形式
2. **必须包含组词列**：表格中**必须**包含「组词」列，组词来自 `assets/top_2500_chars_with_words.json` 的 `words` 字段，帮助孩子联想回忆、辅助判断是否认识该字
3. **组词不得为空**：每一行的组词列**必须**至少包含 1 个词组，用顿号「、」分隔多个词组
4. **汉字列必须加粗**：表格中汉字列使用 `**字**` 加粗格式，方便孩子识别
5. **序号必须层级内全局连续编号（严格执行！）**：序号在**同一层级内跨组连续递增**，不得每组重新从 1 开始。第 1 组为 1-10，第 2 组为 11-20，第 3 组为 21-30，以此类推。用户回复"第N个不认识"时，N 对应该层级内的全局序号

**唯一合法的出题格式模板**：

```
📝 【L{level}·{name}】第 N 组（{globalStart}-{globalEnd} / 共 {testCount} 字）

| 序号 | 汉字 | 组词 |
|:----:|:----:|------|
| 1 | **的** | 好的、是的 |
| 2 | **一** | 一个、一样 |
| 3 | **是** | 但是、总是 |
| ... | ... | ... |
| 10 | **不** | 不要、不去 |

👆 上面哪些字**不认识**？都认识就回复"都认识"
```

**第 2 组序号延续示例**（严格执行跨组连续编号！）：

```
📝 【L1·核心字】第 2 组（11-20 / 共 50 字）

| 序号 | 汉字 | 组词 |
|:----:|:----:|------|
| 11 | **在** | 现在、正在 |
| 12 | **有** | 有点、没有 |
| 13 | **了** | 了解、去了 |
| ... | ... | ... |
| 20 | **地** | 地方、地图 |

👆 上面哪些字**不认识**？都认识就回复"都认识"
```

**禁止出现的格式（反面示例）**：
- ❌ `的 一 是 了 不`（纯文本排列，无表格，无组词）
- ❌ `# 的　一　是　了　不`（大字展示，无表格，无组词）
- ❌ 只有「序号｜汉字」两列而缺少「组词」列的表格
- ❌ 组词列为空或显示"—"的表格行

## ⛔ 熔断检查（最高优先级，不得跳过）

> **🚨 绝对规则：每收到一次用户回复后，必须先完成本节全部步骤，才能决定下一步动作。违反此规则视为严重 Bug。**

### Step 4.1: 状态追踪（必须输出）

收到用户回复后，**必须先在回复中输出以下状态块**，再决定下一步：

```
📋 状态追踪：
- 当前层级：L{level}·{name}
- 本批出题：序号 {globalStart}-{globalEnd}，{N} 字（{字1}、{字2}、...）
- 用户回复：{解析结果}（注意：用户回复的序号为层级内全局序号）
- 逐字处理：{字1}({认识/不认识}, consecutive={N}) → {字2}(...) → ...
- consecutiveUnknown：{之前值} → {当前值}
- 该层已测/testCount：{tested}/{testCount}
- 该层不认识数：{unknown}
- 🔍 熔断检查 A：consecutiveUnknown({value}) ≥ 5 → {是/否}
- 🔍 熔断检查 B：已测({tested}) ≥ 5 且 错误率({unknown}/{tested}={rate}%) ≥ 80% → {是/否}
- ✅ 结论：{继续出题 / 🔴 触发熔断}
```

### Step 4.2: 逐字更新 consecutiveUnknown

按**出题顺序**逐字处理：
- 认识 → `consecutiveUnknown = 0`（重置）
- 不认识 → `consecutiveUnknown += 1`
- **处理过程中如果 `consecutiveUnknown` 达到 5，立即停止处理剩余字，触发熔断**

### Step 4.3: 强制熔断判定

```
条件 A：IF consecutiveUnknown ≥ 5 → 🔴 立即熔断
条件 B：IF 该层已测 ≥ 5 AND 不认识数/已测数 ≥ 0.8 → 🔴 立即熔断
```

- 未熔断 → 继续下一批出题
- **已熔断 → 禁止出题，立即跳转 Step 5 结果报告**

### Step 4.4: "都不认识"快速路径

当用户回复"都不认识"时：

1. 本批**所有字**标记为 unknown
2. `consecutiveUnknown += 本批字数`（每批至少 5 字）
3. **`consecutiveUnknown` 必然 ≥ 5 → 必须触发熔断**
4. **🚫 绝对禁令：下一条消息只能是熔断通知 + 结果报告，绝对不允许出现下一批题目**

### Step 4.5: 出题前置门控

**每次生成下一批题目之前**，必须先检查：
- IF `consecutiveUnknown ≥ 5` → **禁止出题**，跳转结果报告
- IF 当前层已测 ≥ 5 且错误率 ≥ 80% → **禁止出题**，跳转结果报告

只有两个条件都不满足时，才允许生成下一批题目。
